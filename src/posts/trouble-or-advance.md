距离高考还有10天 ，最近有些失眠，我又回想起了两年前临近高考的日子，那时也是焦虑的睡不着觉，现如今，我的弟弟也要高考了，我没啥好教的，最近也很忙，本来是掉了2.4hz接受器 ，如今显示屏也多了两圈坏点，我现在真的破防了，不知道该说什么。

想了又想，还是总结一下 最近对博客的配置吧 ，终于，博客有了个好的开始 ，不知道 以后会不会做个动态的。

除一开始 ，突然其来的兴趣想做一个网站，后面几天 ，对网页整体的设计也是想破了脑袋，也是借鉴了好几位 博主的 设计， 虽然他们都是经过 icp 备案，我也不想维护 和 耗费 更多的金钱 ，所以先做一个 静态网站， 虽说大家不能通过 网站直接登录 和 评论， 也可以希望大家去加一下我的b站小号，私信我就可以和我聊天了，甚至加上我的qq。

对于网站的设计 ，每个人肯定都有自己的想法 ，而AI 可以帮助我们去实现想法， 其实 我并没有花上很多时间 去写自己的代码，只是不断地和AI沟通 ，让它 来进一步提出整体计划，我选择性地采纳，并让它给出 代码 ，在一步步自己去体验 ，找出bug 然后去改 ，这真是一个 烦人地过程。

我总结了一下Ai 地 vibecoding 要点（如您使用claude code）：
1. 在计划执行之前，为AI开启 **plan mode** ,先确定 AI 修改的方向是什么，不要盲目去改，不一定是你想要的答案。
2. 在你认为做出一个项目时，在你安装了 git 的情况下，**请记得一定要 git 提交一下**，不然你让AI在 恢复上一步的时候 它会回退到最初的版本，接下来 ，你只能让AI在按照你的思路再跑一遍，这样会不必要的浪费你的Token和时间， 请一定一定要记住！
3. AI真的很好用，不要产生 病理性依赖。

接下来，除AI使用之外，我想继续分享我的 网络调试过程（我太执拗了）。一开始只是想在github 直接配置一个静态网页，可是网速太慢了，最后尝试了购买CDN、自己的域名,最后还是把它转移到 vercel上，在用 cloudfare 做中装站，这样虽然响应的速度不快，但是，它足够安全以及稳定。
